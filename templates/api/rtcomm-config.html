RtcommConfig

Description

Handles the configuration and settings for the Angular-rtcomm library


Methods

setProviderConfig(config)

Sets the configuration for the EndpointProvider

Parameters

Param Type Details
config object
  • server {string} - MQTT Server which will handle signaling sessions
  • port {integer} - Port number
  • rtcommTopicPath {string} - MQTT Subscription path. Default is '/rtcomm/'
  • createEndpoint {boolean} - Automatically create an RtcommEndpoint. Default is false.
  • appContext {string | optional} - App context
  • presenceTopic {string | optional} -

Example


var app = angular.module('app', ['angular-rtcomm']);

app.controller('Ctrl', function($scope, RtcommConfig){

        var config = {
                "server" : "localhost" ,
                "port" : 1883 ,
                "rtcommTopicPath" : "/rtcomm/",
                "broadcastAudio" : true,
                "broadcastVideo" : true,
                "createEndpoint": true,
                "userid": "",
                "presenceTopic": "demo"
        }

        $scope.config = function(){

                RtcommConfig.setProviderConfig(config);
        }

});

getProviderConfig()
Returns the configuration for the EndpointProvider

getWebRTCEnabled()
Returns true if WebRTC is enabled, false otherwise. Default is always true

getChatEnabled()
Returns true if 'Chat' through text messages feature is enabled, false otherwise. Default is always true

getBroadcastAudio()
Returns true if Audio is enabled, false otherwise. Default is always true

getBroadcastVideo()
Returns true if Video is enabled, false otherwise. Default is always true

isRtcommDisabled()
Returns true if Rtcomm is enabled, false otherwise.
