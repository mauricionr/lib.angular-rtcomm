Getting Started

This repository contains the Rtcomm Angular.js module. The Rtcomm Angular module exposes a set of real-time communication features in the form of a service and a number of Angular directives and controllers. This module is built on the rtcomm.js open source JavaScript library which wraps WebRTC with call signaling and a number of advanced real-time communications capabilities like presence and chat.

Requirements

Rtcomm utilizes MQTT for call signaling so at a minimum you will need an MQTT message broker that supports web sockets. There are many open source and productized versions of MQTT message brokers available. A geat option is to download the WebSphere Liberty Server which includes an embedded MQTT broker and a number of backend Rtcomm services such as call queueing. Download the WebSphere Liberty server free for development and the beta version which includes support for MQTT here.

Install

To install the angular-rtcomm module for use in an AngularJS Application, you can use bower:

bower install angular-rtcomm

Usage

To use the module, you will need to include the following js and css files in your project:

Include angular-rtcomm files: 

<script src="bower_components/angular-rtcomm/dist/angular-rtcomm.js></script>
<link href="bower_components/angular-rtcomm/dist/css/angular-rtcomm.css" rel="stylesheet">

Module Injection

Inject the Angular-Rtcomm module into your angular app or module:

var rtcommLandingPage = angular.module('rtcomm.angular.page', ['angular-rtcomm']);

Quick Configuration

For a quick configuration one can use the RtcommConfigController to setup the angular-rtcomm module for use.


<html ng-app="rtcomm.angular.page">
	<-- Stylesheets, meta and javascript -->
	<head>
	</head>
	
	<body>
		<-- Your webpage content -->
		
		<-- Initialize the Rtcomm library with a JSON configuration file -->	
		<div ng-controller="RtcommConfigController" ng-init="init('rtcommConfig.json')"></div>
	<body>
</html>

An example of rtcommConfig.json can be found here: 


{
        "server" : "localhost" ,
        "port" : 1883 ,
        "rtcommTopicPath" : "/rtcomm/",
        "broadcastAudio" : true,
        "broadcastVideo" : true,
        "createEndpoint": true,
        "userid": "",
        "presenceTopic": "demo"
}

Here we setup parameters for the server, port, the topic path. One could also enable if audio or video should be broadcasted. While the user id could be left empty for later assignment either by Single Sign-On or the RtcommRegister directive. For more information about configuration and setup parameters for the angular-rtcomm module refer to the full api reference.
