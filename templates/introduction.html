Overview

The module exposes a set of real-time communication capabilities in the form of an Angular service and a number of Angular directives and controllers. This module is built on the rtcomm.js open source JavaScript library which wraps WebRTC with call signaling and adds a number of advanced real-time communications capabilities like presence and chat.

Requirements

At a minimum, this module requires an MQTT broker for call signaling. The module is designed to work with any standard MQTT broker that supports the protocol version 3.1 and higher. Some of the advanced capabilities of this module require the WebSphere Liberty server. These will be pointed out in the documentation below. If all you need is basic peer-to-peer calling and presence any open MQTT broker should do.

To make this module as easy to consume as possible, we are providing a backend broker along with an instance of WebSphere Liberty that can be used for testing. It's the same service that is used to demonstrate the directives on this page. For those wishing to create their own sandbox for testing, you can download a developer version of the latest WebSphere Liberty server here which includes everything you need to get started. This server is free for developers and can be up and running in a manner of minutes.

Install

To install the angular-rtcomm module for use in an AngularJS Application, you can use bower:

bower install angular-rtcomm

Usage

To use the module, you will need to include the following JS and CSS files in your project:

Include angular-rtcomm files: 

<script src="bower_components/angular-rtcomm/dist/angular-rtcomm.js></script>
<link href="bower_components/angular-rtcomm/dist/css/angular-rtcomm.css" rel="stylesheet">

It's best to use the versions of JavaScritp libraries installed via bower but if you prefer these can be included from CDNs as follows:

<link href="//maxcdn.bootstrapcdn.com/bootstrap/3.3.1/css/bootstrap.min.css" rel="stylesheet">
<link href="//cdn.rawgit.com/wix/angular-tree-control/master/css/tree-control-attribute.css" rel="stylesheet" type="text/css">

<script src="//ajax.googleapis.com/ajax/libs/jquery/2.1.1/jquery.min.js"></script>
<script src="//maxcdn.bootstrapcdn.com/bootstrap/3.3.1/js/bootstrap.min.js"></script>
<script src="//ajax.googleapis.com/ajax/libs/angularjs/1.3.5/angular.js"></script>

Module Injection

Inject the Angular-Rtcomm module into your angular app or module:

var rtcommLandingPage = angular.module('rtcomm.angular.page', ['angular-rtcomm']);

Quick Configuration

There are a couple of ways to configure the rtcomm-module:

  • Passing a JSON object to the module via the setConfig command.
  • Using the RtcommConfigController to specify a JSON file to load on startup

The easiest way is to use the RtcommConfigController. Here's how: 



<html ng-app="rtcomm.angular.page">
	<-- Stylesheets, meta and javascript -->
	<head>
	</head>
	
	<body>
		<-- Your webpage content -->
		
		<-- Initialize the Rtcomm library with a JSON configuration file -->	
		<div ng-controller="RtcommConfigController" ng-init="init('rtcommConfig.json')"></div>
	<body>
</html>

Open up your text editor and create a file titled 'rtcommConfig.json' in your project directory (double check the path to the files). You can copy the example of rtcommConfig.json which uses the details of our public server setup for use with this module: 


{
        "server" : "localhost" ,
        "port" : 1883 ,
        "rtcommTopicPath" : "/rtcomm/",
        "broadcastAudio" : true,
        "broadcastVideo" : true,
        "createEndpoint": true,
        "presenceTopic": "demo"
}

The JSON contains setup parameters for the server, port, topic path, etc. One could also enable if audio or video should be broadcasted. For more information about configuration and setup parameters for the angular-rtcomm module click here
